Summing node output → wire caps → shared drive PZTs (all plates in parallel).

### 3. Debugging the Signal Path (Runs 161715–210333)

**Problem 1 — PicoScope capture timeout:** Trigger source 5 didn't auto-fire. Fixed by setting trigger source=0 (Ch A).

**Problem 2 — Buffer overrun at 2-channel:** With both channels enabled, ps2000 2204A limits to 2048 samples. `run_block` silently returns 0 for 8064 requests. Fixed by disabling Ch B and using full 8064 single-channel.

**Problem 3 — AWG signal too weak:** Initially connected AWG through R3 (1 kΩ), which combined with 600 Ω internal impedance gave too much attenuation. Removed R3, connected AWG direct to summing node J45.

**Problem 4 — T-connector sinking AWG current:** BNC T-connector to Ch B was shunting AWG signal to the 1 MΩ input, reducing drive by ~50%. Removed T-connector entirely for initial debugging.

**Problem 5 — 0 IM products (reading electrical summing node):** Ch A was wired directly to the summing node, so it read the electrical superposition (which is perfectly linear) rather than the acoustic response of the glass. No nonlinearity possible.

**Solution:** Use the relay mux (Arduino Nano #1, `/dev/cu.usbserial-11310`) to route the selected plate's RX (sense) PZT → Ch A. This measures the acoustic response after the wave has propagated through glass and mixed nonlinearly.

**Problem 6 — Relay mux newline:** `relay_mux.py` sends commands without `\n`, but the firmware requires it. Created `RelayMuxNL` subclass that appends `\n` to all commands.

### 5. Phase 0: Auto-Scan + Balance (Diary V5 Insight Applied)

Applied the key insight from the April 8 Boolean compute experiment: **filter first, then threshold.** Added Phase 0 candidate frequency scan:

1. Drive each of 14 census eigenmodes on AWG at 400 mVpp reference
2. Measure plate acoustic response at each frequency via relay mux
3. Apply geometric-mean threshold (V5 method) → 8 strong modes
4. Greedily pick 3 well-separated modes (min 4 kHz gap)
5. Assign strongest-responding to DDS (weak sources), weakest to AWG
6. Estimate AWG amplitude to balance received carrier levels

**Selected carriers:** DDS#1=30000 Hz, DDS#2=41900 Hz, AWG=47800 Hz at 109 mVpp.

### 7. Phase-Coherent Measurement — T-Connector on Ch B (Run 222329)

Installed BNC T-connector: AWG OUT → one leg to summing node J45, other to PicoScope Ch B. Added `--phase-ref` flag:

- Enables Ch B capture (AWG reference waveform)
- Buffer drops to 2048 samples/ch (ps2000 2-channel limitation)
- Computes per-bin phase stability: φ-lock = |mean(exp(j·Δφ))| across captures
- Genuine signals: φ-lock ≈ 1.0; noise: φ-lock ≈ 0.0
- Phase coherence boost: bins with φ-lock > 0.7 pass IM detection at lower SNR threshold (1.15× instead of 1.3×)
- Compensated with higher averaging: 50 kernel / 48 prescan (vs 20/24 without)

**Phase coherence confirmed:**

| Bin type | φ-lock range |
| -------- | ------------ |
| Carriers | 0.99–1.00    |
| IM prods | 0.03–0.35    |
| Noise    | ~0           |

**11 IM products detected** (up from 7 without phase ref). The phase coherence boost caught marginal bins that magnitude-only missed.

## What We Learned

### A. The Signal Path Matters More Than Drive Power

Reading the electrical summing node (runs 161715–210333) gave a perfectly linear kernel (alignment=1.0) regardless of carrier frequencies or AWG power. The nonlinearity is in the **acoustic propagation through glass**, not in the electrical mixing. The relay mux → sense PZT path is essential.

### B. Carrier Balance Maximizes IM Product Visibility

The 6× imbalance between AWG (1007× SNR) and DDS boards (127–201×) masked weak IM products. Balancing to 1.46× ratio (by reducing AWG to 109 mVpp) exposed 7 IM products. The V5 "filter first" insight directly applied: scan candidate modes, keep only the strong ones, assign strongest to the weakest source.

### C. Phase Coherence Is a Sensitive IM Discriminator

The T-connector on Ch B provides a phase reference that detects genuine mixing products even when their magnitude is only 1.15× above noise. Carriers lock at φ = 0.99–1.00; IM products at 0.03–0.35; noise at ~0. This caught 4 additional IM bins that magnitude-only missed.

### D. Product NMSE Beats Linear — First Proof of Useful Nonlinear Compute from 3-Source Kernel

At product NMSE = 1.50 (linear baseline = 1.92), the glass plate computes multiplicative functions 22% better than a linear model. Combined with 85.7% parity (0% linear), this confirms genuine nonlinear kernel computation from acoustic wave mixing.

### E. The 2048-Sample Tradeoff Is Worth It

Two-channel mode reduces samples from 8064 → 2048 (4× shorter window, ~382 Hz resolution). But carriers are 5900+ Hz apart, so resolution is not the limiting factor. The phase information from Ch B more than compensates.

## Analysis: The Temporal Memory Gap

### What We Have vs What We Need

The 3-source kernel experiment achieved strong **spatial nonlinear computation**: 85.7% parity, 11 IM products, product NMSE beating linear by 22%. This proves genuine acoustic wave mixing in glass plates.

But the original motivation for adding AD9833 DDS boards was not just spatial nonlinearity — it was to **decouple drive from capture** so the PicoScope could run in capture-only mode at **431 Hz** (2.3 ms/step), giving meaningful temporal memory through mode ringdown.

### The Temporal Memory Problem — Recap

From April 17–18 experiments:

| Metric                      | Measured                  | Required (5% retention) | Required (32% retention) |
| --------------------------- | ------------------------- | ----------------------- | ------------------------ |
| Best mode τ                 | 1.99 ms (29.9 kHz, Q=218) | —                       | —                        |
| Step rate (AWG+capture)     | 58 Hz (17 ms/step)        | —                       | —                        |
| Retention at 58 Hz          | **0.02%**                 | ≥ 5%                    | ≥ 32%                    |
| Step rate for 5% retention  | —                         | **~400 Hz** (2.5 ms)    | —                        |
| Step rate for 32% retention | —                         | —                       | **431 Hz** (2.3 ms)      |

The ps2000 firmware blocks the ADC for ~12 ms after any `set_sig_gen_built_in` call, creating a **hard 58 Hz ceiling** when using the internal AWG for frequency hopping.

### What the AD9833 Boards Were Supposed to Fix

With external DDS boards, the **frequency changes happen via SPI to Arduino** (< 1 ms round-trip), bypassing the ps2000 DDS entirely. The PicoScope then runs in capture-only mode at its native 431 Hz rate.

**But today's experiment doesn't use this architecture for temporal memory.** Today's kernel experiment:

- Uses all 3 sources simultaneously (CW, not hopping)
- Measures spatial nonlinear mixing (combinational, not sequential)
- Captures static spectra at each binary ON/OFF pattern
- Has no notion of timestep ordering or input history

### What's Missing: A Temporal NARMA Experiment Using External DDS

To get temporal memory working, we need a **new experiment mode** that:

1. **Sequences input steps** — at each timestep t, encode the NARMA input u(t) as a frequency or amplitude on one or more DDS channels
2. **Changes only via DDS SPI** — never calls `set_sig_gen_built_in` during the sequence
3. **Captures in tight loop** — `run_block` + `ready` + `get_values` only, achieving ~431 Hz
4. **Reads ringdown overlap** — the key physics: mode energy from step t−1 persists into step t's capture window, encoding the prior input in the current spectrum
5. **Uses the same relay mux acoustic path** — essential for reading genuine plate physics, not electrical feedthrough

### Specific Gap Analysis

| Requirement                    | Status                        | Blocker                                             |
| ------------------------------ | ----------------------------- | --------------------------------------------------- |
| External frequency generation  | ✅ AD9833 boards working      | —                                                   |
| SPI frequency changes (< 1 ms) | ✅ Arduino serial `F1:<freq>` | ~50 ms serial round-trip may be too slow            |
| Capture-only PicoScope loop    | ❌ Not implemented            | Need tight capture loop without sig_gen calls       |
| NARMA input encoding           | ❌ Not designed               | How to map u(t) → DDS frequency/amplitude?          |
| Ringdown overlap readout       | ❌ Not tested                 | Does τ=1.99 ms give meaningful carryover at 431 Hz? |
| Arduino command latency        | ⚠️ Unknown                    | Serial at 9600 baud + 50 ms sleep = bottleneck?     |

### The Critical Bottleneck: Arduino Serial Latency

The DDS controller currently uses **9600 baud serial** with a 50 ms sleep after each command. At 9600 baud, a 12-character command like `F1:30000\n` takes ~12 ms to transmit. With the 50 ms sleep, that's **62 ms per frequency change** — _slower_ than the ps2000 DDS stall.

**To achieve 431 Hz step rate (2.3 ms/step), the DDS update must complete in < 0.3 ms.** Options:

1. **Increase baud rate** to 115200 (1 ms per command) and reduce sleep to 1 ms → ~2 ms total. Tight but possibly viable.
2. **Pre-load patterns** — send the entire NARMA input sequence to Arduino, then trigger a timed playback where Arduino autonomously steps through frequencies while PicoScope captures independently.
3. **Direct SPI from host** — bypass Arduino entirely, use FTDI SPI adapter from Mac to AD9833. Sub-millisecond updates guaranteed.

### The Temporal Memory Roadmap

**Phase A — Validate ringdown at 431 Hz (capture-only test):**
Drive a CW tone via DDS, capture at 431 Hz loop, turn DDS off mid-sequence, measure how many capture frames retain the tone energy. This proves/disproves that τ=1.99 ms gives usable memory at 2.3 ms steps.

**Phase B — Fast DDS stepping:**
Increase Arduino baud to 115200, minimize serial round-trip. Or implement pre-loaded sequence playback on Arduino with timer-interrupt driven frequency steps.

**Phase C — NARMA-10 with external DDS:**
Encode u(t) as DDS carrier frequency (e.g., map [0,1] → [28000, 32000] Hz). Step at 431+ Hz. Read spectral features from each capture. Feed to ridge regressor. Compare to software ESN baseline (0.197 NMSE). Target: plate-only < 0.5 NMSE (any temporal memory visible), plate+ESN < 0.17 NMSE (beat April 19 best).

**Phase D — Multi-carrier temporal:**
Use all 3 DDS channels to encode u(t), u(t−1), u(t−2) simultaneously. IM products encode temporal cross-terms (products of current and past inputs). This would combine today's spatial nonlinear mixing with temporal memory — the holy grail.

### What We Know Works (April 18–19 Evidence)

The `ladder_resid_esn` result (plate residuals → ESN) = **0.171 NMSE** proves the plate computes nonlinear features that complement an ESN. The gap is that the plate currently provides only **spatial** nonlinearity (simultaneous multi-frequency mixing) without **temporal** retention.

The April 19 Volterra v3 config achieved **0.167 NMSE** (beating ESN's 0.187 by 11%) — but this relied on appending raw input windows as software features, which is not a pure plate computation.

**For genuine plate temporal memory, we need the capture-only 431 Hz loop with external DDS. The hardware is now in place; the software integration is the remaining step.**

### 12. Ringdown Validation — Five Approaches, All Negative

Attempted to detect acoustic ringdown at the 425 Hz capture rate. All methods failed due to insufficient DDS signal level:

**Test A — DDS spectral energy after off:**

- DDS baseline: 7631 (noise: 7139) → **1.1× SNR**
- After DDS off: readings fluctuate 380–13907 → random noise, no decay curve
- Verdict: Can't measure ringdown of a signal that's barely above noise

**Test B — AWG frequency switch (high-SNR source):**

- AWG 2Vpp baseline: 24.3M (noise: 5361) → **4542× SNR**
- After switching 29300→47800 Hz: **0.0% retention at all steps** (28–298 ms)
- Reason: AWG signal is electrical feedthrough through summing network → Ch A (via T-connector + shared wiring), not acoustic. Electrical signals don't ring down.

**Test C — AWG off (pure decay):**

- After AWG off: readings at 462–1496 ms show 54–222% of "baseline"
- These values (3753–15417) are in the noise band, not genuine residual

**Test D — Fast lock-in detection (DDS ringdown):**

- DDS lock-in baseline: 26.2 (noise: 5.8) → **4.5× SNR**
- After DDS switch: readings 11–35 at 100–3000 ms → noise band
- "80–127% retention" is actually just noise fluctuating around the noise floor

**Test E — Pump-probe cross-modulation sweep:**

- AWG 47800 Hz probe (lock-in SNR: 1966×), swept DDS1 across 25–70 kHz
- Best result: 53 kHz → **1.3σ** magnitude change (need ≥ 3σ for significance)
- All 46 frequencies below 3σ → **no detectable cross-modulation**

### Diagnosis: SNR Is the Bottleneck, Not Speed

| Signal Path                                          | Lock-in SNR | Usable?                     |
| ---------------------------------------------------- | ----------- | --------------------------- |
| AWG 2Vpp → summing node → Ch A                       | 2074×       | Electrical feedthrough only |
| DDS 600mV → 1kΩ → PZT → glass → PZT → relay → Ch A   | **4.5×**    | Too weak for single-capture |
| DDS pump → glass nonlinearity → AWG probe modulation | **1.3σ**    | Undetectable                |

The **speed problem is fully solved** — 425 Hz with zero DDS overhead. But at 4.5× lock-in SNR, single-capture ringdown detection is impossible. The IM products from the kernel experiment (11 products, 85.7% parity) required **20–50 capture averaging**, which is incompatible with temporal memory where each timestep is unique.

### Root Cause: 1kΩ Summing Resistors

The DDS boards output ~600 mVpp. Through 1kΩ summing resistors driving PZTs with ~1–10 kΩ impedance, only a fraction reaches the transducer. The AWG has no resistor (direct connection) and 2Vpp output — it dominates.

**Removing R1 and R2 (the 1kΩ resistors) is the most impactful next step.** This would let DDS signals reach the PZT at full ~600 mVpp, potentially boosting acoustic SNR from 4.5× to 40–50× (based on the AWG's acoustic response scaling).

## What We Learned (Session 2)

### F. Speed Without Signal Is Useless

Achieving 425 Hz capture rate (30.7% theoretical retention) fulfills the temporal memory timing requirement. But the DDS signal through 1kΩ resistors produces only 4.5× lock-in SNR — indistinguishable from noise in single captures. The ringdown exists physically but is unmeasurable at this SNR. **The problem shifted from timing to signal level.**

### G. The AWG Path Is Not Acoustic

The AWG's 2074× lock-in SNR and zero ringdown after frequency switch confirms it's measuring electrical feedthrough: AWG → T-connector → summing node → shared wiring → Ch A. The acoustic path through glass → PZT → relay mux is separate and much weaker.

### H. 20-Capture Averaging vs Single-Capture Detection

The kernel experiment's 85.7% parity required 50-capture averaging per pattern. Temporal memory requires single-capture features at each timestep. This sets a minimum single-capture SNR threshold of ~10× for reliable spectral feature extraction. Current DDS at 4.5× fails this threshold by ~2×.

## The Water-Cooler Version: What We've Actually Proven

_A layman's summary of what the CWM project has demonstrated and where it's headed._

### What Are We Doing?

We're using ordinary glass — the same stuff in your windows — as a computer. Not a screen or a case for a computer. The glass itself does the computing. We send sound waves into it, and the way those waves bounce around, interfere with each other, and ring like a bell _is_ the computation. No transistors, no electricity doing the math. Just physics.

### What Have We Proven So Far?

**1. Glass remembers things.**

Each piece of glass has a unique "fingerprint" — a set of natural vibration frequencies (eigenmodes), like how every wine glass rings at a different pitch when you tap it. We enrolled the fingerprints of 4 glass rods and 5 glass plates, then asked "which glass is this?" by playing back the vibration pattern. **100% correct identification, every time, across hundreds of trials.** The glass plates are so distinctive that even two different spots on the _same_ plate give different fingerprints.

**2. Glass can do logic.**

We stored vibration patterns in two rods (call them Rod A and Rod B), then asked the glass to compute AND, OR, and XOR — the fundamental building blocks of all digital logic. Not by running a program, but by which frequencies rang and which stayed quiet when we drove both patterns simultaneously. **100% correct on all three operations.** The glass computes Boolean logic through the physics of resonance.

**3. Glass can search.**

We asked: "Here's a mixed-up query pattern — which stored pattern is it closest to?" The glass answers by resonating more strongly at frequencies matching the closer pattern. Across 11 interpolation steps from Rod A to Rod B, the glass correctly ranked similarity **100% of the time** (Kendall τ = 1.000). This is nearest-neighbor search — the core operation in AI vector databases — done at the speed of sound.

**4. Glass amplifies neural networks.**

We connected a glass plate to a software Echo State Network (a type of recurrent neural network). The plate's nonlinear acoustic features, combined with the ESN, achieved **0.171 NMSE on NARMA-10** — beating the ESN alone (0.197). The glass isn't just storing data; it's computing nonlinear transformations that make the AI more accurate. And it does this using **17,170× less energy** than the equivalent digital computation.

**5. Glass mixes frequencies like a nonlinear processor.**

When we drive three different ultrasonic tones into a glass plate simultaneously, the plate doesn't just vibrate at those three frequencies. It creates _new_ frequencies — intermodulation products — at the sums and differences of the inputs. We detected **11 intermodulation products** that only exist because the glass is mixing the waves nonlinearly. This is the physical equivalent of multiplying inputs together, which is exactly what neural networks need to do. A linear system can't do this; glass can, and **it beats a linear model by 22%** on multiplicative computation tasks.

### What Are We Working On Right Now?

We're trying to prove glass has **temporal memory** — that the ringing from one input persists long enough to overlap with the next input, so the glass naturally combines current and past information. This is what makes a system a _reservoir computer_: a physical device that automatically mixes present and past inputs through its own dynamics.
