## What We Learned

### A. The AWG + Preamp Path Works

The preamp is not dead and the AWG can drive the current setup. Even at 10 mVpp source drive, the readout produced strong carrier-bin separation from the AWG-off baseline. This is the first successful preamp-in-circuit dataset after the April 24 diagnosis that receive-side amplification was the critical path.

### B. More Drive Is Worse After a Point

The 2 Vpp run clipped. The 100 mVpp run was cleaner but still too dominated by direct feedthrough. The 10 mVpp run gave the most interpretable low-drive operating point. This matches the April 9 and April 23 theme: good computation often appears after balancing source levels, not by maximizing amplitude.

### C. Carrier-Bin Ratio Is Not Enough

The candidate scan ratios looked large at many frequencies, but the RMS and local sweep were broadly flat. That means we were mostly measuring a strong driven tone path, not a sharply selected mechanical mode. This is not a failure; it is the control result telling us the measurement is too close to the source.

### D. The Current Dataset Is Strong for Windowing, Weak for Resonance Discovery

The time-window result is clean. The resonance/Q result is not. We should keep this dataset as the first AWG/preamp uncertainty capture, not as a modal characterization dataset.

### E. We Need to Save More Raw Data Per Candidate

The first version of `tools/awg_preamp_uncertainty.py` saved the final uncertainty frames and local sweep frames, but not every raw candidate scan/noise frame. That is now fixed for future runs: the script was patched after today's run to save raw noise frames, raw scan frames, frame min/max/near-clip stats, and top off-carrier residual bins.

## Is There a Better Way to Capture These Datapoints?

Yes. The better method is a **paired full-spectrum residual capture**, not a carrier-bin scan.

The old script asked: "At the frequency I drove, how much bigger is the bin than AWG-off?" That is a useful sanity check, but it is exactly the question April 24 taught us to distrust. The better question is:

> After subtracting electrical/source leakage, which off-carrier spectral bins change reproducibly, and do those changes differ by read channel?

That captures the thing we actually care about: acoustic redistribution into the glass eigenmode landscape.

### Proposed Capture Protocol: AWG/Preamp Residual Census

For each candidate drive frequency and mux channel:

1. Capture an electronics baseline.
   - Preferred: preamp input terminated, AWG physically disconnected or scope closed.
   - If no rewiring: keep the current AWG-off capture but label it as a weak control, not true silence.

2. Capture a feedthrough control.
   - AWG connected to the drive wiring, but drive PZT mechanically/electrically disconnected or replaced with a dummy load.
   - This measures source leakage into the receive/preamp path.

3. Capture the acoustic condition.
   - Drive PZT connected, RX path through mux and preamp.
   - Same AWG amplitude, same frequency, same capture count.

4. Compute residual spectra.
   - `log(acoustic / feedthrough)` across the full spectrum.
   - Exclude the carrier bin and a guard band around it, e.g. +/-750 Hz.
   - Rank off-carrier bins by reproducible ratio, not by raw magnitude.

5. Compare mux channels.
   - A true acoustic response should have channel-specific structure.
   - Pure electrical feedthrough should look similar across mux channels.
   - Use mux contrast as a second score: `abs(log(mux5 / mux6))` on residual bins.

6. Repeat in randomized interleaved order.
   - Do not capture all OFF then all ON; drift will masquerade as signal.
   - Use blocks like OFF -> FEEDTHROUGH -> ACOUSTIC, shuffled by drive frequency.

7. Run a small amplitude ladder.
   - Suggested: 2, 5, 10, 20, 50 mVpp.
   - If everything scales linearly and stays carrier-locked, it is mostly feedthrough.
   - If off-carrier modes emerge, saturate, or differ by mux, that is the plate.

### Better Selection Metric

Replace "best drive = largest carrier-bin ratio" with:

```
score = off_carrier_residual_energy
      + mux_contrast_bonus
      + reproducibility_bonus
      - carrier_leakage_penalty
      - clipping_penalty
```

This is the same philosophical move as April 18 cross-detection and April 13 cross-relay scoring: extract structure from the whole physical response, not one convenient bin.

### Better Time-Frequency Dataset

The next uncertainty-style run should measure window/FWHM on two targets:

1. The driven carrier, as a DSP control.
2. The strongest reproducible off-carrier residual bin, as the acoustic target.

Then compare them against a synthetic Hann-windowed sine of the same duration. If the acoustic residual is no wider than the synthetic control, the measurement is window-limited. If it is broader, wandering, or multi-peaked, that is physical structure worth investigating.

### What Not to Try Yet

Do not try to use the Pico AWG for immediate ringdown Q. The ps2000 signal-generator update stall is about 12 ms, which is longer than the decay time of the modes we care about. For true ringdown/temporal memory we need one of:

| Option                                            | Why It Helps                                                           |
| ------------------------------------------------- | ---------------------------------------------------------------------- |
| Replacement DDS modules                           | External frequency changes avoid Pico AWG stalls.                      |
| Analog gate / MOSFET switch on AWG output         | Lets us cut drive without calling `set_sig_gen_built_in`.              |
| External function generator with burst/gate input | Gives immediate drive-off timing.                                      |
| Modular v3 patch panel                            | Lets us move drive/read roles and insert dummy loads/controls quickly. |

## Postscript: Pre-Move Residual Verification

Before moving the hardware setup, we ran the patched residual-aware capture twice more at the same 10 mVpp AWG drive:

| Dataset                                                        | Pico Range | Best Script Pick                      | Range Health                                             | Residual Finding                                                                |
| -------------------------------------------------------------- | ---------- | ------------------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------- |
| `data/results/awg/awg_preamp_uncertainty_20260506_113049.json` | +/-100 mV  | mux 6, 49.6 kHz, 55.38x carrier ratio | 34/34 rows had at least one near-rail or max-rail sample | 256/256 top off-carrier bins aligned with integer harmonics of the drive.       |
| `data/results/awg/awg_preamp_uncertainty_20260506_113628.json` | +/-200 mV  | mux 5, 11.4 kHz, 23.74x carrier ratio | 0/34 rows clipped or hit the rail                        | 256/256 top off-carrier bins still aligned with integer harmonics of the drive. |

The 100 mV repeat showed that the earlier high carrier ratios were partly inflated by the preamp output sitting too close to the positive rail. The 200 mV repeat removed that range artifact: no overflow flags, no near-clip samples, and max absolute ADC count stayed around 20,055 rather than 32,767.

The physical interpretation did not improve after fixing the range. The off-carrier residuals were reproducible, but they were harmonic residuals: 2f, 3f, 4f, 10f, 12f, etc. In the clean 200 mV run, the best row at 11.4 kHz had its strongest residuals around 34.2 kHz and 136.8 kHz, i.e. 3f and 12f. The local sweep also remained broad rather than resonant: 9.6% peak-to-peak variation across the 9.9-12.9 kHz sweep and an apparent Q of only about 4.9.

Verdict: **the better capture method worked as a falsification tool**. It showed that the current temporary setup is excellent for confirming AWG/preamp continuity and classical window-width behavior, but it is not yet producing clean acoustic eigenmode residuals. The next useful hardware move is justified: use the modular v3 wiring to add feedthrough/dummy-load controls, reduce DC offset at the preamp/scope input, and make drive/read role changes easier.

### Residual Character After Ch B Cancellation

The off-carrier bins were still mostly harmonics:

| Signal                   | Harmonic-aligned top bins |
| ------------------------ | ------------------------- |
| Raw Ch A                 | 243 / 256 = 94.9%         |
| Reference-cancelled Ch A | 219 / 256 = 85.5%         |

The Ch B subtraction reduced carrier/feedthrough energy, but it did **not** reveal a clean set of non-harmonic acoustic eigenmode residuals. The strongest cancelled residuals still clustered around 2f-type harmonics, especially for the higher drive candidates.

### Can We Run Heisenberg Tests Now?

Yes, but with a narrower claim.

At 2048 samples, the maximum observation window is only 2.621 ms, so the longest-window linewidth floor is about 668-715 Hz. That is enough to demonstrate the finite-window time-frequency uncertainty trend, but not enough to resolve narrow mechanical linewidths.

For the full reference-cancel run at mux 6 / 68.2 kHz:

| Signal                   | 256 samples | 512 samples | 1024 samples | 2048 samples | Verdict                                                                                |
| ------------------------ | ----------- | ----------- | ------------ | ------------ | -------------------------------------------------------------------------------------- |
| Raw Ch A                 | T\*dF 1.125 | T\*dF 1.750 | T\*dF 1.750  | T\*dF 1.875  | Usable finite-window control.                                                          |
| Ch B reference           | T\*dF 1.500 | T\*dF 1.750 | T\*dF 1.750  | T\*dF 1.750  | Best clean reference for the Heisenberg/window test.                                   |

Verdict: **we can run the Heisenberg-style finite-window test now with two channels**, using Ch B as the reference/control and raw Ch A as the preamp path. We should not yet claim a Heisenberg-style result on a clean acoustic residual, because the cancelled residual remains harmonic/feedthrough dominated.

## Bottom Line

Today validated the AWG + preamp path and produced a clean time-window spectral-width dataset. It did **not** produce a clean mechanical resonance or Q measurement. The better next capture is a full-spectrum residual census with explicit feedthrough controls, off-carrier scoring, mux-channel contrast, randomized interleaving, and raw-frame preservation.

That is exactly the kind of protocol the earlier diaries have been pushing us toward all along: listen to the whole spectrum, compare across channels, preserve controls, and never confuse a convenient analysis artifact for the physics.
