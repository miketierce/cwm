### 4. Hardware Associative Recall — First Attempts

Built `tools/associative_recall_hw.py` — a 4×4 discrimination matrix experiment. For each rod's enrolled frequency pattern (query), drive AWG at those frequencies and measure all 4 rods' responses via relay mux.

**Run 1 (shared baseline, all-equal scoring): 25% accuracy**
- Rod 2 dominated all queries regardless of query pattern
- Root cause: different PZT coupling per relay path

**Run 2 (per-relay baselines): 25% accuracy**
- Per-relay baseline normalization didn't help
- Baseline frequencies themselves hit other rods' resonances through shared drive
- Ratios often <1.0 (baseline higher than "on-resonance" signal)

**Run 3 (uniqueness-weighted scoring): 0% accuracy**
- Rod 4 relay path has highest gain, dominates all weighted scores
- Weighting amplified the coupling asymmetry problem

### 5. The Breakthrough: Template Matching

**Offline analysis** of stored per-peak data revealed the solution. Tested three approaches on existing JSON data without re-running hardware:

| Scoring Method                     | Accuracy |
| ---------------------------------- | -------- |
| Cross-relay normalization          | 1/4 (25%) |
| Unique-peaks-only                  | 2/4 (50%) |
| **Template matching**              | **4/4 (100%)** |

**How template matching works:**

For each query frequency, normalize raw magnitude across all 4 sense relays (cross-relay normalization). Then:
- **Boost +3×** when a sense rod has an enrolled peak within 3% of the query frequency (expected resonance)
- **Penalize −1×** when the sense rod has no enrolled peak near the query frequency (spurious response)

The diagonal always wins because ALL query frequencies match the query rod's own enrolled peaks (gets boosted at every frequency), while off-diagonal rods only match at overlapping frequencies and get penalized at the rest.

**Key insight:** This doesn't need amplitude enrollment, re-enrollment, or even good baselines. It only uses frequency positions we already have + cross-relay signal normalization.

## What We Learned

### A. Baseline-Ratio Scoring Is Fundamentally Broken for Shared-Drive Arrays

In a shared-drive topology, "off-resonance" baseline frequencies inevitably excite resonances of *other* rods. The baseline is not a true feedthrough measurement — it includes cross-rod contamination. Per-relay baselines don't fix this because the contamination comes from the shared drive side, not the sense side.

### B. Cross-Relay Normalization Is the Key Pre-Processing Step

Normalizing magnitude across all 4 sense relays at each frequency cancels relay-path gain differences. The resulting fractions (rod's share of total response) are purely about which rod resonates, not which relay has better coupling.

### C. Template Matching Converts CIM from "Maybe" to "Definite"

The enrolled frequency positions aren't just for identification — they serve as a **decoder ring** for associative recall. By knowing which frequencies *should* cause each rod to resonate, we can score expected vs spurious responses. This is the physical analog of a matched filter in signal processing.

### D. 80% Spectral Overlap Is Not a Barrier

Rods 1 and 4 share 80% of their peaks, yet template matching distinguishes them with +2.6 margin. The remaining 20% unique peaks, amplified by the boost/penalize scoring, are sufficient for discrimination.

### E. Stuck AWG Drive Is a Real Hazard

After one run, Rod 3 had a continuous drive signal stuck on it. Required explicit AWG shutdown + relay off. Added to the cleanup procedure in `_close_scope()`.
