CWM Temporal Reservoir — Wiring Guide
Arduino Nano + AD9833 DDS + 8-CH Relay + PicoScope 2204A

Overview

This guide covers the solder-free wiring for the CWM temporal reservoir proof-of-concept. Phase 1 uses binary ON/OFF drive via the AD9833 DDS (7 new jumper wires). Phase 2 adds analog amplitude control via an MCP4921 DAC (6 additional wires).

Wiring Table (7 jumper wires)

WireFromToColor SuggestionNotes
1 AD9833 VCC Arduino 5V Red Power — from breadboard 5V rail
2 AD9833 GND Arduino GND Black Ground — from breadboard GND rail
3 AD9833 DAT (SDATA) Arduino D11 (MOSI) Blue SPI data — shared bus
4 AD9833 CLK (SCLK) Arduino D13 (SCK) Yellow SPI clock — shared bus (also LED pin)
5 AD9833 FSY (FSYNC) Arduino D10 Orange SPI chip-select for AD9833
6 AD9833 OUT TX PZT bus (+) Sine output → exciter PZTs in parallel (replaces PicoScope AWG)
7 AD9833 GND TX PZT bus (−) Signal return — same ground as PZT leads
D13 shared: Arduino D13 is both the onboard LED and SPI SCK. The LED will flicker during SPI traffic — this is normal. After SPI.begin(), D13 is fully dedicated to the clock.

4. Phase 2 Wiring — MCP4921 Amplitude Control

Insert the MCP4921 between the AD9833 output and the relay COM input. The AD9833 sine feeds into VREF; the DAC scales it by a 12-bit code (0–4095).

MCP4921 DIP-8 Pinout

         ┌────╥────┐
  VDD  1 │●   ║    │ 8  VOUT  → relay COM input
   CS  2 │         │ 7  VSS (GND)
  SCK  3 │ MCP4921 │ 6  VREF  ← AD9833 OUT
  SDI  4 │         │ 5  LDAC
         └─────────┘

Wiring Table (5 new jumper wires + 1 reroute)

WireFromToColor SuggestionNotes
8 MCP4921 pin 1 (VDD) Arduino 5V Red Power
9 MCP4921 pin 7 (VSS) Arduino GND Black Ground
10 MCP4921 pin 2 (CS) Arduino D9 Green SPI chip-select (separate from AD9833 CS on D10)
11 MCP4921 pin 3 (SCK) Arduino D13 (SCK) Yellow Shared SPI clock bus
12 MCP4921 pin 4 (SDI) Arduino D11 (MOSI) Blue Shared SPI data bus
13 MCP4921 pin 5 (LDAC) GND Black Tie low = output updates immediately on CS rising edge

Signal Reroute for Phase 2

ConnectionPhase 1Phase 2
AD9833 OUT → TX PZT bus (wire 6) → MCP4921 pin 6 (VREF)
MCP4921 pin 8 (VOUT) — → TX PZT bus (replaces wire 6)
Phase 2 uses Arduino D9: This pin was Relay IN8 (Plate 5 NW-RX). If you need all 8 relays and the DAC, move relay 8 to pin A0 and update the firmware. For the temporal proof, relay 5 (Plate D NE) is the primary target, so D9 is available.
MCP4921 multiplying-DAC physics check:
Vout = VREF × D / 4096
Max signal frequency: 95 kHz → required slew rate 0.179 V/µs
MCP4921 output slew rate: 0.55 V/µs — 3× headroom, no distortion
SPI update: ~1 µs at 8 MHz — negligible vs 2.3 ms capture step

6. Step-by-Step Assembly

Phase 1 — AD9833 DDS (Binary ON/OFF)

  1. Seat the AD9833 breakout module in the breadboard (any free rows).
  2. Connect VCC → 5V rail and GND → GND rail (wires 1–2).
  3. Run DAT → D11 (wire 3 — SPI MOSI).
  4. Run CLK → D13 (wire 4 — SPI SCK).
  5. Run FSY → D10 (wire 5 — chip select).
  6. Run AD9833 OUT → TX PZT bus (+) — the wire pair that drove exciter PZTs (wire 6).
  7. Run AD9833 GND → TX PZT bus (−) — signal return (wire 7).
  8. Disconnect PicoScope AWG BNC from TX PZT bus (AD9833 replaces it).
  9. Flash relay_controller_dds.ino to the Arduino Nano via Arduino IDE.
  10. Verify: open serial monitor at 9600 baud, send F29925 — expect DDS:29925.

Phase 2 — MCP4921 Amplitude Control

  1. Seat the MCP4921 DIP-8 across the breadboard center channel (pin 1 = top-left).
  2. Connect pin 1 (VDD) → 5V rail and pin 7 (VSS) → GND rail.
  3. Connect pin 5 (LDAC) → GND rail (auto-update mode).
  4. Run pin 2 (CS) → Arduino D9 (new CS line).
  5. Run pin 3 (SCK) → D13 (shared SPI clock bus).
  6. Run pin 4 (SDI) → D11 (shared SPI data bus).
  7. Move wire 6: AD9833 OUT → pin 6 (VREF) instead of TX PZT bus.
  8. Run pin 8 (VOUT) → TX PZT bus (amplitude-scaled sine output).
  9. Flash updated firmware with MCP4921 support, test amplitude sweep.

8. Verification Checklist

  • AD9833 module seated in breadboard, no bent pins
  • 5V and GND connected (check with multimeter if unsure)
  • D10 (FSYNC), D11 (MOSI), D13 (SCK) wired to correct AD9833 pins
  • AD9833 OUT + GND wired to TX PZT bus (exciter leads)
  • PicoScope AWG BNC disconnected from TX PZT bus
  • PicoScope Ch A BNC still connected to relay COM output (RX path)
  • Firmware flashed: serial monitor shows OK:0 on boot
  • Send F29925 → responds DDS:29925
  • Send 5 → responds OK:5 (Plate D NE relay)
  • PicoScope shows sine wave at ~29.9 kHz on Ch A
  • Send Foff → signal disappears
  • Run temporal_proof.py — warm-up magnitude > 100

9. Troubleshooting

SymptomLikely CauseFix
No DDS:xxxx response Old firmware still on Nano Flash relay_controller_dds.ino
DDS:29925 but no signal on scope AD9833 OUT not connected to TX PZT bus Check wire 6 (OUT → TX PZTs) and wire 7 (GND return)
Signal on scope but wrong frequency AD9833 module has non-standard MCLK Check crystal marking: should be 25 MHz. If different, update DDS_MCLK in firmware
Warm-up magnitude < 100 Wrong relay selected or PZT disconnected Verify relay 5 = Plate D NE, check PZT leads
SPI not working (no frequency change) D11/D13 wired to wrong AD9833 pins DAT=D11 (MOSI), CLK=D13 (SCK), FSY=D10 — check all three
Relay clicks but no signal path TX PZT wiring reversed or disconnected AD9833 OUT → TX PZT (+), AD9833 GND → TX PZT (−)
