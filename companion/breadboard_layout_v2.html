Board Allocation

FULL Board A — Analog Preamp

OPA2134PA, gain resistors, bypass caps, 9V batteries. Only analog receive path. Nothing else touches this board.

FULL Board S — Source Generation

Arduino 2 (DDS SPI master), 3× DDS modules, 8-ch relay (source selector: routes AWG or any DDS to drive output). All signal sources live here.

HALF Board M — Read Mux

Arduino 1, 16-ch relay module. Selects which RX PZT connects to preamp. Mux common exits as twisted pair to Board A.

HALF Board D — Drive Distribution

AWG BNC input, drive PZT output, dummy-load header. Physical breakout between source board and the cassette.

HALF Board X1 — Expansion

Spare for additional RX sensors, second preamp, or extra relay channels as the experiment grows.

HALF Board X2 — Expansion

Spare. Can hold a second mux bank, additional DDS, or reference/calibration hardware.

FULL Board A — Analog Preamp

Full-size 830 tie-point. Columns 1-63, rows A-E (top half) / F-J (bottom half). OPA2134PA straddles center gap, notch left, pins on columns 28-31.

Board A is the ONLY board with analog gain.No AWG, DDS, Arduino, or relay wires here.

FULL Board S — Source Generation

Full-size 830 tie-point. Houses Arduino 2 (DDS SPI), up to 3 DDS modules (AD9833 or similar), and the 8-channel relay used as a source selector.

DDS modules share SPI bus with individual CS lines.8-ch relay selects which source reaches the drive output.

Source Selector (8-ch Relay)

Relay Ch Input Source Use Case
1 AWG (from Pico tee) Default fallback; reference-cancel via Ch B tee.
2 DDS 1 output Primary DDS drive. Replace AWG when DDS boards arrive.
3 DDS 2 output Second frequency or phase reference.
4 DDS 3 output Third frequency for multi-tone experiments.
5 Combined DDS 1+2 Summed output via resistor network for dual-tone.
6 Dummy load only Silence drive for baseline/control measurements.
7-8 Spare Future: external source, calibration signal.
Switching rule: Only one relay channel ON at a time for single-source drive. For multi-DDS simultaneous, use the resistor summing node on Ch 5 and route multiple DDS outputs in parallel.

DDS SPI Wiring (Arduino 2)

Arduino Pin Signal Destination
D13 SCK All 3 DDS CLK (shared bus)
D11 MOSI All 3 DDS DATA (shared bus)
D10 CS1 DDS 1 FSYNC
D9 CS2 DDS 2 FSYNC
D8 CS3 DDS 3 FSYNC
D2-D7 Relay IN1-IN6 8-ch relay control inputs
5V VCC DDS VCC + relay VCC
GND GND DDS GND + relay GND (local to Board S)
Ground isolation: Board S GND is Arduino USB ground. Do NOT connect Board S GND to Board A AGND. They are separate ground domains.

The ONLY wire from Board M to Board A: Mux common signal + mux common return as a short twisted pair. Signal goes to Board A hole A20. Return goes to Board A top blue rail near C24. No other Board M wire touches Board A.

HALF Board D — Drive Distribution

Half-size board. Physical breakout point between Board S source selector output and the drive PZT on the cassette. Also provides BNC tee point for Ch B reference and dummy-load header.

Purpose: Board D is just a physical breakout. When using DDS instead of AWG, the selected source from Board S 8-ch relay COM goes here. When using AWG, the AWG tee branch goes to Board S input first (so it can be selected by the relay alongside DDS options).

Ground Domain Map

Analog Ground Domain

Board A only.

Star point: C24 on Board A.

Sources: battery midpoint, Pico Ch A shield, mux return (from Board M).

Isolated from USB ground.

Digital Ground Domain 1

Board M (Arduino 1 + 16-ch relay).

Powered by Arduino 1 USB.

GND shared between Arduino 1 and relay module only.

No wire to Board A GND.

Digital Ground Domain 2

Board S + Board D (Arduino 2, DDS, 8-ch relay, drive distribution).

Powered by Arduino 2 USB.

Board D return = Board S GND via cable #4 return wire.

No wire to Board A GND.

Ground coupling rule: The mux return wire (#7) carries the RX signal return from Board M to Board A star ground. This is the ONLY intentional path where a "digital board" wire touches analog ground. It works because the mux common output is the analog signal itself. Do not add any other ground connections between boards.

DDS Transition & Multi-Source Modes

When the 3 DDS boards arrive, attach them to Board S without disturbing Board A or Board M. The 8-ch relay on Board S lets you switch sources without rewiring.

Mode 1: AWG Only (current)

Board S relay ch 1 ON (AWG selected)
Ch B Direct BNC tee reference
DDS modules Not installed or powered off
Scripts awg_preamp_refcancel_2048.py

Mode 2: Single DDS Drive

Board S relay ch 2 ON (DDS 1 selected)
Ch B Optional: AWG as timing reference, or repurpose for second RX
Advantage Phase-coherent frequency control, no AWG jitter
Scripts New DDS driver + existing capture logic

Mode 3: Multi-DDS Simultaneous

Board S relay ch 5 ON (summed DDS 1+2)
DDS 1 Frequency f1
DDS 2 Frequency f2
Summing Each DDS output through 10k into shared node
Use case Two-tone interference, beat frequency measurement

Mode 4: DDS + AWG Comparison

Procedure Run capture with relay ch 1 (AWG), then ch 2 (DDS 1), same frequency
Compare Residual spectrum should differ if DDS has lower jitter/feedthrough
No rewiring Just relay command change between captures

Attach DDS boards with header pins into Board S breadboard holes. Each DDS breakout is typically 5-6 pins wide. Place them in columns 18-22, 18-22 (lower half), and 28-32 as shown on the Board S diagram. Exact placement depends on your specific breakout board pin spacing.

Bring-Up Checklist & Test Sequence

Phase 1: Board A Power-Off Checks

  • OPA pin 8 (F28) → +9V rail. Confirm with meter.
  • OPA pin 4 (E31) → −9V rail. Confirm with meter.
  • C24 → top blue rail (star ground jumper in place).
  • E24 → F24 bridge jumper in place.
  • 100nF from D28 to D24 (close to pin 8).
  • 100nF from B31 to B24 (close to pin 4).
  • 10µF +9V bulk cap: + to red rail, − to blue rail.
  • 10µF −9V bulk cap: + to blue rail, − to purple rail.
  • Rf 100k from C28 to C29.
  • Rg 10k from D29 to D24.
  • RIN 4.7k from A20 to A30.
  • RBIAS 1M from C30 to C24.
  • ROUT 47Ω from A28 to A40.
  • Park jumper H29→H30 (OPA B follower).
  • Park jumper H31→H24 (OPA B +IN to GND).
  • No AWG/DDS/Arduino/relay wires on Board A.

Phase 2: Power-On Meter Checks

Test Expected
+9V rail to blue rail +8 to +9.6V
−9V rail to blue rail −8 to −9.6V
+9V to −9V 16 to 19V total
OPA output (A40) with input grounded Near 0V (within ±50mV)
OPA +IN (A30) with nothing connected Near 0V via 1M bias

Phase 3: Signal Tests

Step Setup Expected
1. Input short Jumper A20 to blue rail. AWG off. Ch A quiet, no rail.
2. Layout feedthrough Input still grounded. AWG 10mVpp on. Ch A should show minimal carrier.
3. Mux terminated Board M relay ch 15 (input short). AWG on. Ch A stays quiet.
4. Normal acoustic Board M relay ch 5 or 6. AWG on. Ch A shows signal + preamp gain.
5. Ref-cancel Same as 4 + Ch B reference. Cancellation > 4dB improvement.
6. DDS swap Board S relay ch 2 instead of ch 1. DDS-driven signal, compare to AWG.

Phase 4: Run Command

# AWG mode (current)
python tools/awg_preamp_refcancel_2048.py \
  --mux-channels 5,6 \
  --range-a-mv 200 --range-b-mv 500 \
  --drive-uvpp 10000 \
  --scan-averages 8 --uncertainty-frames 24

# DDS mode (future - script TBD)
python tools/dds_drive_capture.py \
  --dds-freq-hz 85000 \
  --source-relay 2 \
  --mux-channels 5,6
Success criteria after rewiring: (1) No clipping at ±200mV Ch A range. (2) Step 2 feedthrough < −40dB of step 4 signal. (3) After Ch B cancellation, top residual bins are NOT mostly drive harmonics. (4) Mux-channel-specific spectral differences visible.
