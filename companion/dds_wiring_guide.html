CWM — External DDS Wiring Guide
3× AD9833 + 3× MCP4921 + Arduino Nano + Resistive Summing Network

Rev 1 — 2026-04-19  |  Print: Cmd+P → PDF or paper

1. Purpose

Decouple frequency generation from the PicoScope AWG. Three AD9833 DDS modules generate carrier frequencies, three MCP4921 DACs control their amplitudes (via resistive attenuation), and a summing network combines all outputs into a single drive signal for the plate piezo. The PicoScope runs in capture-only mode at 431 Hz step rate.

3. System Signal Flow

Amplitude control approach: Each MCP4921 DAC output (0–4.096V) is used as a variable voltage divider reference. The AD9833 output (~600mVpp) passes through R_sum (1kΩ) to the summing junction. The MCP4921 doesn't directly modulate the DDS output — instead, we use it as a simple on/off gain control by feeding its output to an analog multiplier point. For the initial experiment, the DACs set a DC amplitude scale factor that the Arduino firmware applies digitally by scaling the DDS output enable. Simplest path: just use the AD9833 outputs directly at ~600mVpp each, and modulate amplitude by rapidly toggling the DDS output on/off (SLEEP mode). The MCP4921s give finer control later.
Simpler first pass (recommended): Skip the MCP4921 DACs initially. Drive all 3 AD9833 outputs at fixed amplitude through the summing resistors. Encode NARMA input values as frequency selections rather than amplitude. This tests the temporal memory hypothesis (frequency hopping at 431 Hz) without the complexity of analog amplitude control. Add DACs in pass 2 once temporal memory is confirmed.

GND Ground GND rail

CLK SPI SCK (clock) D13 (shared)
FSY SPI CS (active LOW — frame sync) D10 / D9 / D8 (one per module)
Output: The GY-9833 module has an analog output pad/pin (sometimes labeled OUT or VOUT) on the board. It's a ~0.6Vpp sine wave centered at ~VCC/2. Check your specific module — some have a 3-pin header (VCC, GND, OUT) on one side and the 5-pin SPI header on the other.

Pin conflict check: The relay board uses D2–D6 + A0–A3. If you want both boards connected simultaneously, shift the DAC CS pins to A0–A2 instead of D5–D7, OR disconnect the relay board during DDS experiments. For initial testing, just use the DDS board alone.

7. Breadboard Layout (Physical Placement)

Reading the layout: The board is shown vertically (long axis down). Left side (cols a–e) has the three AD9833 DDS modules stacked. Right side (cols f–j) has the three MCP4921 DACs. The summing network at the bottom collects all DDS outputs through 1kΩ resistors to a single node. Power rails (+/−) run the full length at top and bottom edges — bridge both + rails and both − rails together.

8. Step-by-Step Wiring Instructions

Step 1: Power Rails

  • Connect Nano 5V pin → breadboard + rail (red)
  • Connect Nano GND pin → breadboard − rail (black)
  • Bridge both + rails together (top and bottom) with a jumper
  • Bridge both − rails together (top and bottom) with a jumper

Step 2: AD9833 Modules (×3)

For each AD9833 module, insert the 5-pin header into the breadboard and wire:

  • VCC → + rail (5V)
  • GND → − rail
  • DAT → Nano D11 row (shared MOSI bus wire)
  • CLK → Nano D13 row (shared SCK bus wire)
  • FSY → Nano CS pin (D10, D9, or D8)

Bypass capacitors (per module):

  • 0.1µF ceramic cap between VCC pin row and GND pin row (as close to module as possible)
  • 10µF electrolytic cap between VCC pin row and GND pin row (observe polarity: + stripe to VCC)

Step 3: MCP4921 DACs (×3)

Insert each DIP-8 straddling the center gap. Wire:

  • Pin 1 (VDD) → + rail
  • Pin 2 (CS) → Nano CS pin (D7, D6, or D5)
  • Pin 3 (SCK) → Nano D13 row (shared)
  • Pin 4 (SDI) → Nano D11 row (shared)
  • Pin 5 (LDAC) → − rail (GND — enables immediate latch)
  • Pin 6 (VREF) → + rail (5V reference = 5V full scale)
  • Pin 7 (VSS) → − rail
  • Pin 8 (VOUT) → leave free for now (amplitude control output)

Bypass capacitor (per DAC):

  • 0.1µF ceramic cap between pin 1 (VDD) and pin 7 (VSS) — directly across the chip

Step 4: Shared SPI Bus

SPI bus tip: Use a single continuous wire for each bus, bending it to touch each module's row. Or use short jumpers daisy-chained between adjacent modules. Keep bus wires short and close to the board — long wires pick up noise at MHz SPI speeds.

Step 5: Summing Network

  • Insert 4× 1kΩ resistors, each with one leg in the row of its signal source
  • All four opposite legs go to the same row (the summing node)
  • Run a wire from the summing node to the exciter piezo positive lead
  • Exciter piezo negative lead → GND rail
PicoScope AWG connection: BNC center conductor (signal) → R4 → summing node. BNC shield (ground) → GND rail. The AWG outputs the static 10-carrier probe waveform, set once at the start and never changed during the run.

Step 6: PicoScope Capture

  • PicoScope CH-A remains connected to the pickup piezo via existing relay mux
  • No changes to the capture side — same BNC connections as before

9. Complete Wiring Schematic (Text)

10. Bypass Capacitor Placement (Critical)

Electrolytic polarity matters! The 10µF caps have a white/gray stripe indicating the NEGATIVE leg. That stripe goes toward GND. If reversed, the cap can pop (loudly). The 0.1µF ceramic caps have no polarity — either orientation is fine.

11. Testing Sequence (Before Running Experiments)

  1. Power test: Connect Nano USB. Measure 5V between + and − rails with multimeter. Should read 4.8–5.1V.
  2. Single DDS test: Upload firmware that sets AD9833 #1 to 1 kHz. Probe the OUT pin with PicoScope CH-A. Should see clean 1 kHz sine, ~600mVpp.
  3. All DDS test: Set #1=19kHz, #2=29.9kHz, #3=53.9kHz. Verify all three on scope.
  4. Summing test: Connect all three through summing resistors. Scope at summing node should show composite waveform. FFT should show 3 distinct peaks.
  5. DAC test: Write 2048 to MCP4921 #1 (midscale). Measure VOUT with multimeter — should read ~2.5V.
  6. Combined test: Add PicoScope AWG (static 10-carrier signal) through R4. Verify all 13 frequency peaks in FFT at pickup.

12. Common Mistakes to Avoid

  • Forgetting LDAC tie-down: MCP4921 pin 5 MUST be connected to GND. If left floating, the DAC output never updates.
  • Swapped MOSI/SCK: AD9833 labels are DAT/CLK (not the Arduino labels). DAT=MOSI, CLK=SCK.
  • All CS lines LOW at once: Firmware must keep all CS pins HIGH by default, pull ONE low at a time to address that device.
  • Electrolytic backwards: Stripe = negative. Will pop if reversed.
  • Missing ground bridge: If using both power rails, bridge the − rails together.
  • SPI mode: AD9833 uses SPI Mode 2 (CPOL=1, CPHA=0). MCP4921 uses SPI Mode 0 (CPOL=0, CPHA=0). Firmware must switch modes between addressing DDS and DAC.
